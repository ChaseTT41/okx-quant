1. **趋势商品论 (飞天猪)**: 选风口而非选个股

## 四、三层防火墙审批

### L1: 🏛️ 安检门 (Market Regime Classifier)
**文件**: `market_regime.py`
**输入**: F&G指数、BTC趋势、波动率、Alpha方向
**输出**: 市场环境(5级) + 操作偏好 + 杠杆上限 + 最大风险

| F&G | BTC趋势 | → 判定 | 操作偏好 | 最大杠杆 |
|-----|---------|--------|---------|---------|
| 0-20 | 跌 | 极度看空 | short_only | 3x |
| 0-20 | 涨 | 中性 | neutral | 5x |
| 21-40 | 跌 | 温和看空 | short_only | 5x |
| 21-40 | 涨 | 中性 | any | 5x |
| 41-60 | 涨 | 温和看多 | any | 10x |
| 61-80 | 涨 | 极度看多 | long_only | 12x |
| 81-100 | 涨 | 温和看多 | long_only | 5x |

**黑名单**: F&G<15 + BTC跌 + 高波动 = 全面禁止交易
**规则引擎**: 20+条优先级规则，通配符匹配，命中即停

### L2: ⚖️ 共识投票 (Strategy Consensus Engine)
**文件**: `trade_gates.py` → `StrategyConsensusEngine`
**规则**:
- 至少 **2个独立策略** 方向一致
- ML动量(0.35) + Alpha(0.35) 权重最高
- 平局(同权反向) → 强制HOLD
- 视角多样性加分: 技术+价值+量化 三视角共识 > 单一视角多策略

**共识等级**:
| 等级 | 分数 | 含义 |
|------|------|------|
| strong | ≥45 | 高确定性，多视角一致 |
| medium | ≥30 | 中等确定性，可执行 |
| weak | ≥20 | 弱信号，需更多确认 |
| none | <20 | 无共识 |

### L3: 🔴 执行官 (Execution Officer)
**文件**: `trade_gates.py` → `ExecutionOfficer`
**6项必检清单**:
1. ✅ 市场环境方向匹配 (安检门 action_bias 校验)
2. ✅ ≥2个策略达成共识
3. ✅ 单笔风险 ≤ 5% 账户总额 (Chase哥指定, 止损对标仓位保证金)
4. ✅ 持仓集中度 ≤ 上限
5. ✅ 无黑天鹅事件 (F&G≤15时禁止)
6. ✅ 综合置信度 ≥ 45 (medium) / ≥ 55 (strong)

**Check 3 风险计算 (v3.3 修正: 止损对标账户, 非标的价格)**:
```
max_loss = 账户权益 × 5%           # 单笔最多亏这么多
stop_pct = max_loss / 名义价值      # 反推标的止损距离%
stop_pct = clamp(0.5%, 15%)        # 太紧是噪音, 太宽没意义

示例 ($50账户):
  AAPL: 10x, 名义$197 → stop=1.3% → 亏$2.50=账户5%  ✅
  LUNR: 5x,  名义$100 → stop=2.5% → 亏$2.50=账户5%  ✅
```

**置信度计算**:
```
base = consensus_score
quality_bonus = min(15, high_quality_count × 5)
diversity_bonus = (unique_perspectives - 1) × 8  # 技术/价值/量化
final = min(100, max(0, base + quality_bonus + diversity_bonus))
```

## 五、仓位与风控规则

### 仓位管理
| 权益范围 | 最大持仓数 | 单笔最大仓位 | 集中火力 |
|---------|-----------|------------|---------|
| $0-100 | 2个 | 40-50% | 至少40%, 留一半对冲 |
| $100-200 | 3个 | 25-40% | 至少25% |
| $200-500 | 4个 | 15-30% | 至少15% |
| $500+ | 5个 | 3-5% | tier 原值 |

### 止损规则 (v3.3: 对标账户, 非标的价格)
> 💰 **单笔最大亏账户 5%** — Chase哥指定, 止损对标仓位保证金

- **止损公式**: `max_loss = 权益 × 5%` → `stop_pct = max_loss / 名义价值`
- **clamp**: 最少 0.5% (太紧是噪音), 最多 15%
- 止损不是惩罚，是保护。基于账户总风险而非标的波动
- 随着盈利扩大，逐步上移止损（追踪止损，待实现）
- 止损位不固定，可以移动但只能往有利方向移

### 止盈规则 (v3.3: 布林上轨+ATR动态)
> 🎯 **止盈基于技术面, 不用死板公式** — 2026-06-23 修复

- **股票**: `止盈 = 布林上轨距离 × 1.2`, 用ATR调整, clamp(5%, 20%)
- **加密**: 仍用杠杆引擎档位建议 (加密波动大, 20-50%合理)
- 例: AAPL $299, BB上轨$299 → TP≈+8%=$323 (而非旧公式+40%=$412!)
- 止盈为动态值, 持有期间根据新行情调整
- **关键: 下单前就想好你要持有多久、预期涨到哪**

### 杠杆映射 (v4)
```
blended_wr → 杠杆 / 仓位 / 止损 / 止盈(建议)
  ≥75% → 20x  / 2.0%  / -10% / +30%
  65-75% → 15x / 3.0%  / -10% / +30%
  55-65% → 10x / 3.5%  / -12% / +40%
  50-55% → 5x  / 4.0%  / -15% / +50%
  <50%  → 1x  / 5.0%  / -20% / +80%  (跳过不开)
```
公式: `blended_wr = 0.6 × 历史胜率 + 0.4 × ML confidence ± 幅度修正 ± 费率修正`

### 止盈止损计算公式
```
止损价(做多) = 入场价 × (1 - 0.10)  # ≥10%
止损价(做空) = 入场价 × (1 + 0.10)
止盈价 = 入场价 × (1 + 预期涨幅%)   # 动态!
```

## 六、入场保护机制 (v3.1 新增)

### 🛡️ 价格位置校验 (`check_price_position()`)
**文件**: `leverage_engine.py`

开仓前自动检查价格在24h区间的位置，避免追高杀跌：
- **做多**: 价格 > 24h区间70%分位 → 拒绝 (追高风险)
- **做空**: 价格 < 24h区间30%分位 → 拒绝 (杀跌风险)
- 数据来源: OKX REST API ticker (high24h, low24h)

### 🔍 裸仓检测 (`check_all_positions_sl_tp()`)
**文件**: `leverage_engine.py`

启动时和每轮扫描周期遍历所有实盘持仓，检查每个仓位是否有活跃的 SL/TP algo 订单保护。
- 发现裸仓 → 立即告警，补设止盈止损
- 返回 `{all_protected, naked_positions, protected_positions}`

### ✅ SL/TP 订单验证 (`verify_algo_order_live()`)
**文件**: `leverage_engine.py`

开仓后逐个 algoId 确认 `state=live`（最多重试2次，间隔1秒）。
不能只依赖 POST 返回的 code==0，必须确认订单真正激活。
验证失败 → 告警 "仓位裸奔" → 日志记录。

### ⚡ API 断路器
**文件**: `auto_trade_daemon.py`

连续3次 API 错误触发熔断，降级为模拟盘模式。
- 冷却期: 20分钟
- 熔断期间: 不执行任何实盘交易
- 恢复: 冷却结束后自动重新尝试

## 七、完整交易流程

### 阶段 1: 扫描 (每10分钟)
```
1. run_all_strategies() → K线技术面信号 (OKX REST 日线)
2. MultiPerspectiveScanner.scan_all() → 权哥 + 西蒙斯 + Serenity
3. 合并所有信号 → 送入量化总部
```

### 阶段 2: 审批 (每笔交易)
```
L1 安检门: F&G→regime判定 → 方向过滤 → 杠杆/风险上限
    ↓ (pass)
L2 共识投票: 按symbol分组 → 多策略投票 → 视角多样性加分
    ↓ (≥2策略同意, ≥medium共识)
L3 执行官: 6项检查 → 最终置信度 → 仓位确认
    ↓ (6/6通过)
🛡️ 价格位置: check_price_position() → 追高杀跌检测
    ↓ (通过)
✅ 可执行
```

### 阶段 3: 执行
```
1. 确定仓位大小 (在审批阶段已定)
2. 设定止损 (≥10%)
3. 设定止盈 (基于预期判断，可动态调整)
4. 创建OKX合约单 (REST API)
5. 创建SL/TP algo订单 (REST API)
6. verify_algo_order_live() 验证SL/TP已激活
7. 记录到交易日志
```

### 阶段 4: 持仓监控
```
- 每30分钟: check_all_positions_sl_tp() 检查所有持仓保护
- 追踪止损: 盈利>15%后上移止损到成本价 (待实现)
- 时间止损: 持仓4小时未盈利→评估是否平仓 (待实现)
- 权益回撤>10% → 停止新开仓
- API断路器: 连续错误→自动降级
```

## 十、📋 交易报告标准模版

### 10.1 报告结构

每次开仓/平仓/调仓讨论，按以下四要素输出：

#### 🎯 做什么方向
- **明确方向**: 做多 / 做空 / 观望
- **标的优先级排序**: 哪个最值得做、哪个次之
- **具体价位**: 入场价 / 止损价 / 止盈目标价
- **仓位大小**: 建议仓位百分比

#### ❓ 为什么做
- **驱动逻辑一句话总结**: 这一单的核心矛盾是什么
- **当前市场核心矛盾**: 是什么在推动价格
- **为什么不是反向操作**: 做多→为什么不做空（反之亦然）
- **置信度**: 高/中/低，附具体数值

#### 📊 理由 — 四维交叉验证
每个维度必须有**具体数据支撑**，禁止泛泛而谈：

| 维度 | 要点 | 数据要求 |
|------|------|---------|
| 🏦 **宏观/资金面** | 利率/流动性/资金流向 | F&G指数、加息预期、ETF资金流、爆仓数据 |
| 📉 **技术面** | K线形态/量价/指标 | 支撑阻力位、成交量变化、RSI/MACD/布林带 |
| 😱 **情绪面** | 市场情绪/散户行为 | F&G、资金费率、OI变化、社交媒体热度 |
| ⚖️ **监管/事件面** | 政策/新闻/事件 | 期权到期日、监管政策、项目升级/解锁 |

#### 💡 给自己的建议
- **仓位管理**: 这次用多少仓位、为什么
- **风险控制**: 止损设哪里、最大亏多少
- **离场条件**: 什么情况提前走（不等到止损）
- **加仓条件**: 什么情况可以加
- **时间框架**: 预期持有多久

## 十一、⚡ 杠杆与持仓周期规则 v3.5 🆕

> Chase哥指令 2026-06-25: 短线确定趋势后可以拉高杠杆，最高20倍。

### 11.1 核心原则

**杠杆高低取决于两个因素：趋势确定性 × 持仓周期。**

| 条件 | 杠杆 |
|------|------|
| 趋势不明确 | **不开仓**，杠杆再低也不开 |
| 趋势明确 + 长线(>1d) | 1-3x |
| 趋势明确 + 中短线(1h-4h) | 5-10x |
| 趋势明确 + 短线(5m-30m) | **15-20x** 🔥 |
| ❌ 任何情况 | **禁止超过20x** |

### 11.2 趋势确认标准

开高杠杆前必须满足以下全部条件：

1. **安检门方向明确** — action_bias 为 short_only 或 long_only（非 any）
2. **多TF一致** — 5m/15m/1h/4h 至少3个时间框架方向一致
3. **F&G配合** — 极端恐惧做空 / 极端贪婪做多
4. **裸K结构确认** — 高低点序列符合趋势方向（LH+LL=下跌，HH+HL=上涨）

> 🚨 四项缺一不可。有任何一项不满足 → 降杠杆至5-10x或不开仓。

### 11.3 短线高杠杆风控

| 参数 | 短线(15-20x) |
|------|-------------|
| 止损 | **入场价的1.5-2.5%**（紧止损） |
| 持仓时间 | 5-30分钟，最长不超过1小时 |
| 仓位占比 | 不超过权益的**30%**（含杠杆） |
| 单笔最大亏损 | ≤账户的**5%** |
| 趋势反转信号 | **立即平仓**，不扛单 |

### 11.4 为什么短线可以用高杠杆？

- 短线持仓时间短（<30min），市场突变概率低
- 紧止损（1.5-2.5%）× 高杠杆（20x）= 单笔风险30-50%，但仓位仅占账户30% → 实际风险 = 账户的9-15% × 仓位占比 → 控制在5%以内
- 关键公式: `实际风险 = 仓位% × 杠杆 × 止损%` → 必须 ≤ 5%
- 例: 25%仓位 × 20x × 2.5%止损 = 12.5% → ⚠️超标！→ 仓位降至10%: 10% × 20x × 2.5% = 5% ✅

### 11.5 杠杆递进规则

- 首次开仓: 用最低杠杆（短线10x起）
- **浮盈加仓**: 第一单盈利后，第二单可加至15x
- **连胜加码**: 连续3单盈利 → 可升至20x
- **亏损降级**: 任何一单止损 → 下笔降回10x，重新爬坡
